# Refactoring Workflow

Use this workflow whenever code quality improvements are requested.

## Objectives

Improve code quality without changing application behavior.

## Process

1. Understand the existing implementation.
2. Identify duplicated logic.
3. Identify overly complex code.
4. Simplify where possible.
5. Preserve functionality.
6. Run tests.
7. Update documentation if required.
8. Update the related ticket.
9. Commit changes.

## Rules

Never introduce unnecessary architecture changes.

Never mix new features into a refactor.

Document significant improvements in the ticket.
