# Release Workflow

Before creating a release:

## Verification

* All tickets completed
* No critical bugs
* Build succeeds
* Tests pass
* Documentation updated
* Version updated
* Changelog updated

## Release Steps

1. Build release.
2. Verify executable.
3. Update version.
4. Update CHANGELOG.md.
5. Commit release.
6. Push to GitHub.
7. Create release package.
8. Store release inside releases/.

## Deliverables

* Executable
* Release notes
* Version number
* Changelog
